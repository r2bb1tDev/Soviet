## 🤔 Что это такое?

**Soviet** — мессенджер без серверов и без регистрации. Никакого email, телефона или аккаунта — только никнейм и пара криптографических ключей.

В отличие от Telegram, WhatsApp или Signal:
- ✅ **Нет центрального сервера** — сообщения идут напрямую между людьми
- ✅ **Нет регистрации** — просто запустил и начал общаться
- ✅ **Нет слежки** — никто не видит твои сообщения, даже разработчики
- ✅ **Работает без интернета** — в одной Wi-Fi сети через LAN-режим
- ✅ **Открытый код** — можешь проверить сам

---

## 💬 Как это работает?

При первом запуске Soviet генерирует твою **пару ключей** (как в Bitcoin):
- **Публичный ключ** — это твой адрес, его можно давать другим
- **Приватный ключ** — хранится на твоём устройстве, никуда не отправляется

Чтобы написать кому-то — нужен его публичный ключ. Поделиться своим можно через QR-код или скопировав приглашение.

### Как передаются сообщения?

Soviet автоматически выбирает лучший способ доставки:

```
1️⃣  LAN (локальная сеть)   — быстро, без интернета, в одной Wi-Fi сети
2️⃣  P2P mesh (интернет)    — напрямую через интернет, без серверов
3️⃣  Nostr relay (fallback) — через публичные relay-серверы, если P2P недоступен
```

Все сообщения зашифрованы — даже relay-серверы не могут их прочитать.

## ✨ Возможности

### Личные сообщения
- 🔒 End-to-End шифрование (ChaCha20-Poly1305)
- ✏️ Редактирование и удаление отправленных сообщений
- 💬 Ответы на сообщения (Reply)
- 😊 Emoji-реакции
- 📎 Файлы и изображения (до 10 МБ)
- ✓✓ Статусы доставки (отправлено / доставлено / прочитано)
- ⌨️ Индикатор "печатает..."

### Группы
- Создание групп до любого размера
- Приглашение по публичному ключу
- Системные уведомления о входе/выходе

### Каналы (Nostr)
- Публичные каналы через протокол Nostr
- Создание и подписка на каналы по ID
- Только создатель может писать в канал

### Контакты
- Добавление по публичному ключу или QR-коду
- Поиск пользователей в сети
- Статусы: онлайн / отошёл / занят / невидимка
- Аватары и статус-текст

### Интерфейс
- 🌙 Автоматическая тёмная/светлая тема
- 🐻 Белый медведь в системном трее
- 🔔 Push-уведомления о новых сообщениях
- 📲 QR-код для обмена контактом

## 🔐 Безопасность

| Компонент | Технология |
|-----------|------------|
| Ключи | Ed25519 (подпись) + X25519 (шифрование) |
| Шифрование сообщений | ChaCha20-Poly1305 |
| Обмен ключами | ECDH X25519 + HKDF |
| Хранение ключей | Системное хранилище ОС (DPAPI / Keychain / libsecret) |
| Протокол сети | Noise Protocol (libp2p) |

Все прямые сообщения шифруются на твоём устройстве. Ни серверы, ни разработчики не имеют доступа к содержимому.

## 🌐 Режимы связи

### 1. LAN (без интернета)
Если ты и собеседник в одной Wi-Fi сети — Soviet найдёт друг друга автоматически через mDNS и передаст сообщение напрямую по TCP.

**Подходит для:** офис, дом, мероприятие, закрытая сеть.

### 2. P2P mesh (интернет)
Прямое соединение через интернет без промежуточных серверов. Soviet использует libp2p Kademlia DHT для поиска собеседников по всему миру.

**Подходит для:** обычное общение через интернет.

### 3. Nostr relay (fallback)
Если P2P не работает (NAT, файервол) — сообщение отправляется через публичные Nostr relay-серверы в зашифрованном виде. Relay видит только зашифрованный blob, но не содержимое.

**Подходит для:** сложные сетевые условия, корпоративные сети.

## 🚀 Быстрый старт

1. Скачай установщик с [Releases](https://github.com/r2bb1tDev/Soviet/releases/latest)
2. Установи и запусти
3. Введи никнейм — ключи создадутся автоматически
4. Нажми **QR-код / Поделиться** в настройках и скинь приглашение другу
5. Друг вставляет приглашение в **"Добавить контакт"** — готово!
