## [2.5.6] — 2026-04-16

**Патч: белое окно и пропажа постов — НАСТОЯЩАЯ причина найдена**

### Исправлено
- **Белое окно попаута + пропажа постов** — корневая причина: в `capabilities/default.json` было `"windows": ["main"]`. Попаут-окна с лейблами `chat-*` и `chan-*` не имели НИКАКИХ разрешений → все `invoke()` вызовы молча падали → данные не загружались → пустой белый экран. Исправлено на `"windows": ["main", "chat-*", "chan-*"]`

---

## [2.5.5] — 2026-04-16

**Патч: macOS CI codesign — окончательный фикс**

### Исправлено
- **CI macOS**: `if: ... && env.APPLE_CERTIFICATE != ''` всегда было `false` — `env.*` в `if`-выражениях ссылается на job/workflow-уровень, а не step-уровень. Исправлено на `secrets.APPLE_CERTIFICATE != ''`
- **CI macOS**: Apple signing vars убраны из `env:` блока tauri-action и теперь попадают в окружение только через `$GITHUB_ENV` внутри шага импорта сертификата. Пустая строка `""` больше не утекает в `codesign`

---

## [2.5.4] — 2026-04-15

**Патч: CI macOS падал из-за пустого APPLE_SIGNING_IDENTITY**

### Исправлено
- **CI macOS codesign** — `APPLE_SIGNING_IDENTITY` теперь передаётся в tauri-action только при наличии сертификата (`APPLE_CERTIFICATE` секрет); раньше пустая строка `""` попадала в `codesign --sign ""` → ошибка "item could not be found in keychain". Аналогично для `APPLE_ID`, `APPLE_PASSWORD`, `APPLE_TEAM_ID` (нотаризация)

## [2.5.3] — 2026-04-15

**Патч: автообновление не перезапускалось + белое окно (окончательный фикс)**

### Исправлено
- **Автообновление не перезапускало приложение** — после `downloadAndInstall()` теперь вызывается `relaunch()` из `@tauri-apps/plugin-process`. Без него обновление скачивалось и устанавливалось, но приложение продолжало работать со старой версией
- **Белое окно при открытии чата/канала** — тема теперь применяется через синхронный inline `<script>` в `<head>` HTML, до загрузки CSS и первого рендера браузера

## [2.5.1] — 2026-04-15

**Патч: CI-фикс · SQLCipher миграция · Белое окно · Тема попаута**

### Исправлено
- **CI Windows build** — заменён `shogo82148/actions-setup-perl@v1` (падал с "unknown distribution: Strawberry") на добавление Strawberry Perl в `$GITHUB_PATH` (уже предустановлен на `windows-latest`)
- **SQLCipher миграция** — детектирование plaintext БД теперь работает через чтение первых 16 байт (magic bytes `SQLite format 3\0`) вместо ненадёжного `Connection::open`; добавлен `PRAGMA key = ""` перед ATTACH, что устраняло потерю данных при обновлении с версий < 2.5.0
- **Белое окно при открытии чата/канала в отдельном окне** — тема теперь применяется синхронно (IIFE при загрузке модуля) до первого рендера через `localStorage('soviet_theme')` вместо async `invoke`
- **Рассинхронизация темы в попаутах** — `App.tsx` теперь сохраняет разрешённую тему в `localStorage` при каждом обновлении, так что ChatPopout всегда читает актуальное значение

## [2.5.0] — 2026-04-14

**SQLCipher · Sybil-защита · Code Signing CI/CD**

### Добавлено
- **SQLCipher — полное шифрование файла БД** — вся база данных (история сообщений, контакты, каналы) теперь зашифрована на диске через SQLCipher с 256-битным ключом; ключ хранится в OS keyring (DPAPI / Keychain / libsecret); автоматическая zero-downtime миграция plaintext → encrypted при первом запуске v2.5
- **Защита от Sybil-атак** — репутационная система P2P-узлов: учёт успешных/неуспешных соединений, бан узлов с постоянными ошибками, снижение скора за подозрительное поведение, восстановление репутации со временем
- **Code Signing в CI/CD** — подпись бинарей Windows (NSIS/MSI) через сертификат в GitHub Secrets; подпись и нотаризация macOS через Apple Developer ID; автоматический импорт ключей в workflows

### Изменено
- GitHub Actions: добавлены шаги импорта Windows PFX и macOS keychain
- `tauri.conf.json`: добавлены `certificateThumbprint`, `signingIdentity`, `providerShortName`

## [2.0.0] — 2026-04-10

**NAT hole-punching · Шифрование БД · Голосовые · Ответы на сообщения**

### Добавлено
- **NAT hole-punching (DCUtR)** — libp2p relay client v2 + dcutr behaviour; подключается к relay-узлам при старте, резервирует circuit-слоты, эмитит `p2p-relay-reserved` / `p2p-dcutr-success`; позволяет устанавливать прямые P2P соединения через NAT/файрвол без ручных настроек
- **Шифрование чувствительных данных БД** — генерируется уникальный 32-байтовый ключ на устройстве и хранится в OS keyring; секретный Nostr-ключ шифруется ChaCha20-Poly1305 перед записью в БД; обратная совместимость со старыми plaintext-значениями
- **Голосовые сообщения** — запись через MediaRecorder API (audio/webm, audio/ogg), отправка как зашифрованный файл, рендеринг через `<audio controls>` прямо в пузыре сообщения, таймер записи в стиле ICQ
- **Ответы на сообщения (Reply)** — правый клик → «Ответить»; превью-полоска над строкой ввода с кнопкой ×; цитата отображается внутри пузыря (2-строчный обрезкой, акцентный бордер слева); `reply_to` сохраняется в БД и передаётся по всему стеку

### Улучшено
- Терминальный UI (зелёный фосфор, CRT scanlines, monospace) — уточнены стили пузырей и compose bar
- Версия приложения в «О приложении» читается динамически через `getVersion()` API

### Добавлено
- **Terminal / MS-DOS UI** — полный рестайл интерфейса: зелёный фосфор (#00FF41), CRT scanline оверлей, monospace шрифт (Share Tech Mono / VT323 / Courier New), квадратные углы, glow эффекты, мигающий курсор

- **Динамическая версия** — в «О приложении» версия теперь читается через `getVersion()` API, не хардкодится

### Исправлено
- **`capabilities/default.json`** — создан файл разрешений Tauri v2 ACL; его отсутствие было главной причиной падения всех предыдущих билдов
- **LAN peer TTL** — пиры теперь удаляются из карты через 90 сек без активности (cleanup-поток каждые 60 сек); раньше offline-пиры висели вечно и блокировали fallback на P2P/Nostr
- **Синхронизация статуса** — hello-пакет с аватаркой отправляется при КАЖДОМ presence-пакете, а не только от новых пиров (фиксит статус после перезапуска)
- **Входящие файлы** — определяется content_type (`image`/`file`) по MIME-типу, plaintext сохраняется для рендеринга без повторной расшифровки
- **Имена разрешений** — исправлено `notification:allow-show-notification` → `notification:allow-show`
