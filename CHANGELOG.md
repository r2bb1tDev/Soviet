## [1.1.0] — 2026-04-03

**Интернет-режим + P2P mesh + улучшения UX**

### Добавлено

#### Интернет-связь (Nostr DM — Kind 4444)
- Зашифрованные личные сообщения через Nostr relay (Kind 4444) как fallback при недоступности LAN/P2P
- Подписка на входящие DM по публичному ключу пользователя (hex-формат, `p` tag)
- Дедупликация входящих сообщений через `message_exists_by_ts()` (защита от повторной доставки relay)
- Поддержка трёх relay: `wss://relay.damus.io`, `wss://nos.lol`, `wss://relay.nostr.band`

#### P2P mesh-сеть (libp2p 0.53)
- Новый модуль `src-tauri/src/p2p/mod.rs` — полноценный libp2p stack
- `SovietBehaviour`: mDNS + Kademlia DHT + Identify + Ping + RequestResponse
- Детерминированный PeerID из Ed25519 ключа Soviet — отдельный обмен ключами не нужен
- Kademlia DHT для обнаружения пиров через интернет (bootstrap: public IPFS nodes)
- mDNS заменяет UDP broadcast для LAN-обнаружения (надёжнее в современных сетях)
- Входящие P2P-сообщения обрабатываются тем же пайплайном, что и LAN
- Команда `get_p2p_peers` — список известных P2P-пиров с адресами и Soviet-ключами
- Транспортная цепочка: LAN TCP → P2P libp2p → Nostr DM relay
- Frontend: секция "🌐 ИНТЕРНЕТ (P2P)" в сайдбаре
- Polling P2P-пиров каждые 15 секунд

#### Поиск пользователей
- Новый компонент `UserSearchModal` — поиск по никнейму, @id или публичному ключу
- Поиск по контактам, LAN-пирам и P2P-пирам одновременно
- Кнопка 🔎 рядом с поиском в сайдбаре
- Метки источника: контакт / LAN / P2P
- Кнопка "+ Добавить" открывает AddContact с предзаполненными данными

#### Настройки — раздел "Сеть"
- Статус libp2p DHT: количество подключённых пиров (бейдж)
- Статус Nostr relay: всегда активен
- Список relay-серверов
- Ваш P2P ID (PeerId) с кнопкой копирования

### Исправлено

- **Onboarding layout**: корень страницы теперь `flexDirection: column` — кредиты "FSOCIETY × LURKHUB" больше не съезжают вбок
- **Импорт ключей**: убрано поле никнейма — при восстановлении аккаунта никнейм автоматически устанавливается как `User_XXXXXXXX` (первые 8 символов ключа). Изменить можно в Настройках
- **ShareCard**: убран системный короткий ID (`ID: XXXXXXXX`) — отображается только custom `@id` если установлен
- **Копирование ключей в Настройках**: кнопки 📋 теперь показывают `✓` на 2 секунды после копирования (публичный и приватный ключи отдельно)

## [1.0.0] — 2026-04-02

**Первый стабильный релиз Soviet**

### Добавлено

#### Архитектура и стек
- Tauri 2.1 (Rust бэкенд + React/TypeScript фронтенд)
- Vite для сборки React
- Zustand для управления состоянием
- SQLite (rusqlite) для локального хранилища

#### Идентичность и криптография
- Генерация Ed25519/X25519 пар ключей при первом запуске
- Хранение приватного ключа в защищённом хранилище ОС (DPAPI/Keychain/libsecret)
- Custom user ID (3-10 символов, уникальный идентификатор)
- Экспорт и импорт ключей для восстановления идентичности
- E2E шифрование ChaCha20-Poly1305 через ECDH X25519
- Подписание сообщений Ed25519

#### Личные сообщения
- Прямые один-на-один чаты с E2E шифрованием
- Редактирование отправленных сообщений
- Удаление сообщений (с уведомлением)
- Emoji-реакции на сообщения
- Ответы (reply/quote) на сообщения
- Форвард сообщений
- Поддержка code blocks (``` фенсированные и ` инлайн) с кнопкой </> для вставки
- Передача файлов до 10 МБ
- Инлайн-отображение изображений
- Индикатор набора текста ("Никнейм печатает...")
- Статусы доставки (✓ отправлено, ✓✓ доставлено, ✓✓ прочитано)

#### Групповые чаты
- Создание групп любым участником
- Приглашение участников по публичному ключу
- Присоединение и выход из групп
- Удаление группы (только создатель)
- Системные сообщения при входе/выходе участников
- E2E шифрование группового канала

#### Nostr каналы
- Создание и присоединение к публичным каналам
- WebSocket relay-подключение (wss://relay.damus.io по умолчанию)
- Kind 40/41/42 (create/update/message) и kind 5 (delete) события
- Метаданные канала: имя, описание (about), аватар (base64)
- Писатель (только создатель может отправлять)
- Счётчик подписчиков
- Удаление канала (только создатель)

#### Локальная сеть (LAN)
- Автоматическое обнаружение соседей через UDP broadcast (порт 7433)
- Прямое TCP-соединение для E2E чатов
- Кастомный JSON-протокол для LAN-пакетов
- Поддержка: hello, message, file, group_invite, group_message, member_left, group_dissolved, reaction, edit, delete, typing, read_receipt, contact_request
- Работа полностью без интернета

#### Контакты
- Добавление контактов по публичному ключу
- Обнаружение контактов в LAN сети
- QR-код для быстрого добавления (формат: Soviet | nickname | pubkey | @customId)
- Share card / invite card
- Списки контактов с сортировкой
- Статусы пользователей: Online / Away / Busy / Offline
- Пользовательский текст статуса
- Аватары (загрузка и хранение base64-изображений)
- Поиск контактов по имени и публичному ключу
- Поиск каналов по имени

#### Интерфейс
- React 18 + TypeScript компоненты
- Hauptfenster с боковой панелью контактов и центральным окном чата
- Всплывающее окно настроек (Settings)
- Экран первого запуска (Onboarding)
- Автоматическая адаптация под светлую/тёмную тему ОС
- Ручное переключение тем в настройках
- Иконки статусов (🟢 Online, 🟡 Away, 🔴 Busy, ⚫ Invisible, ⚪ Offline)
- Системный трей с иконкой белого медведя
- Анимация конвертика на иконке при новых сообщениях
- Контекстное меню трея (статусы, настройки, выход)
- Уведомления ОС о новых сообщениях

#### Безопасность
- E2E шифрование всех прямых сообщений (ChaCha20-Poly1305)
- E2E шифрование групповых чатов (групповой симметричный ключ)
- Подписание всех сообщений (Ed25519)
- Верификация подписей при получении
- SQLite с поддержкой шифрования (не включено в v1.0)
- HKDF для вывода ключей сессии
- Perfect Forward Secrecy (эфемерные ключи для каждого сообщения)

#### Хранение данных
- SQLite база данных (unencrypted в v1.0)
- Таблицы: settings, contacts, contact_requests, chats, messages, group_members, message_reactions, nostr_channels, nostr_messages
- Локальное хранилище (не синхронизируется между устройствами)
- Экспорт контактов
- Импорт контактов

#### Сборка и инсталляция
- Tauri build система для всех платформ
- NSIS инсталлер для Windows
- DMG для macOS (universal binary: Intel + Apple Silicon)
- AppImage и .deb для Linux
- Команда `npm run tauri dev` для разработки
- Команда `npm run tauri build` для продакшена

#### Документация
- README.md — обзор, быстрый старт, инструкции по установке
- docs/ARCHITECTURE.md — архитектура системы (Tauri, модули Rust)
- docs/CONTACTS.md — система контактов, QR-коды, статусы
- docs/CRYPTO.md — криптография (Ed25519, X25519, ChaCha20-Poly1305, ECDH, HKDF)
- docs/LAN_MODE.md — LAN-режим, UDP broadcast, TCP, протокол
- docs/UI_DESIGN.md — дизайн интерфейса, темы, иконки, макеты
- docs/MESH_NETWORK.md — планируемая mesh-сеть (Wi-Fi Direct, Bluetooth)
- protocol/PROTOCOL.md — LAN и Nostr протоколы
- CONTRIBUTING.md — руководство для контрибьюторов
- SECURITY.md — политика безопасности

## Знания об ограничениях в v1.0

- Нет mesh-сети (Wi-Fi Direct, Bluetooth)
- Нет синхронизации между устройствами (каждое устройство — отдельный аккаунт)
- Нет шифрования БД (SQLite unencrypted)
- Нет голосовых сообщений или звонков
- Нет видео-звонков
- Nostr relay фиксирован на relay.damus.io (можно изменить в code)
- Макс размер файла: 10 МБ
- Не синхронизируется история между устройствами
