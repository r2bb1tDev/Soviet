## [2.0.0] — 2026-04-10

**NAT hole-punching · Шифрование БД · Голосовые · Ответы на сообщения**

### Добавлено
- **NAT hole-punching (DCUtR)** — libp2p relay client v2 + dcutr behaviour; подключается к relay-узлам при старте, резервирует circuit-слоты, эмитит `p2p-relay-reserved` / `p2p-dcutr-success`; позволяет устанавливать прямые P2P соединения через NAT/файрвол без ручных настроек
- **Шифрование чувствительных данных БД** — генерируется уникальный 32-байтовый ключ на устройстве и хранится в OS keyring; секретный Nostr-ключ шифруется ChaCha20-Poly1305 перед записью в БД; обратная совместимость со старыми plaintext-значениями
- **Голосовые сообщения** — запись через MediaRecorder API (audio/webm, audio/ogg), отправка как зашифрованный файл, рендеринг через `<audio controls>` прямо в пузыре сообщения, таймер записи в стиле ICQ
- **Ответы на сообщения (Reply)** — правый клик → «Ответить»; превью-полоска над строкой ввода с кнопкой ×; цитата отображается внутри пузыря (2-строчный обрезкой, акцентный бордер слева); `reply_to` сохраняется в БД и передаётся по всему стеку

### Улучшено
- Терминальный UI (зелёный фосфор, CRT scanlines, monospace) — уточнены стили пузырей и compose bar
- Версия приложения в «О приложении» читается динамически через `getVersion()` API

## [1.3.5] — 2026-04-09

**Терминальный UI + OS-уведомления + надёжность LAN + фикс билдов**

### Добавлено
- **Terminal / MS-DOS UI** — полный рестайл интерфейса: зелёный фосфор (#00FF41), CRT scanline оверлей, monospace шрифт (Share Tech Mono / VT323 / Courier New), квадратные углы, glow эффекты, мигающий курсор
- **OS push-уведомления** — `tauri-plugin-notification` вызывается при каждом входящем сообщении (системный всплывающий попап)
- **Аватарка в хедере чата** — если у контакта есть аватарка, показывается изображение вместо буквы
- **Динамическая версия** — в «О приложении» версия теперь читается через `getVersion()` API, не хардкодится

### Исправлено
- **`capabilities/default.json`** — создан файл разрешений Tauri v2 ACL; его отсутствие было главной причиной падения всех предыдущих билдов
- **LAN peer TTL** — пиры теперь удаляются из карты через 90 сек без активности (cleanup-поток каждые 60 сек); раньше offline-пиры висели вечно и блокировали fallback на P2P/Nostr
- **Синхронизация статуса** — hello-пакет с аватаркой отправляется при КАЖДОМ presence-пакете, а не только от новых пиров (фиксит статус после перезапуска)
- **Входящие файлы** — определяется content_type (`image`/`file`) по MIME-типу, plaintext сохраняется для рендеринга без повторной расшифровки
- **Имена разрешений** — исправлено `notification:allow-show-notification` → `notification:allow-show`
