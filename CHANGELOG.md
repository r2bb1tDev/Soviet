## [2.7.0] — 2026-04-19

**5 скинов интерфейса + preview на hover + voice-через-все-каналы**

### Добавлено
- **5 скинов интерфейса** (переключаются в Настройки → Скин, применяются мгновенно):
  - **Terminal** (по умолчанию) — dark-grey Linux-терминал с DOS-шрифтом
  - **ICQ Classic** — классический сине-жёлтый ICQ (светлая тема)
  - **Matrix** — зелёный phosphor на чёрном
  - **Amber CRT** — янтарный CRT ретро
  - **Purple** — современный тёмно-фиолетовый (Telegram-стиль)
  - Хранится в settings как `theme_skin`, применяется через `data-theme-skin` атрибут на `:root`.
- **Tab peek preview** — при ховере на вкладке через 500 мс появляется tooltip с последним сообщением и таймстампом.
- **Voice через все каналы** — голосовые сообщения теперь идут параллельно через LAN + P2P + Nostr (раньше были ограничены одним транспортом).

## [2.6.1] — 2026-04-19

**Power-user вкладки + онлайн-timeline**

### Добавлено
- **Горячие клавиши вкладок**:
  - `Ctrl+Tab` / `Ctrl+Shift+Tab` — следующая / предыдущая вкладка
  - `Ctrl+1..9` — переход к вкладке N
  - `Ctrl+W` — закрыть активную
  - `Ctrl+Shift+T` — переоткрыть последнюю закрытую (как в браузере)
- **Drag-and-drop переупорядочивание вкладок** (нативный HTML5 DnD).
- **Контекстное меню на вкладке** (ПКМ): Перейти / Закрепить / Закрыть остальные / Закрыть.
- **Закрепление вкладок** (pin) с индикатором-точкой, закреплённые не пропадают.
- **Recently closed tabs** — history закрытых чатов с возможностью восстановления (`Ctrl+Shift+T`).
- **Онлайн-timeline в профиле контакта** — sparkline активности за последние 7 дней на основе временных меток сообщений.

### Обоснование
Настоящие multi-window через `std::process::Command` переносятся на v2.7: требует полноценной IPC-архитектуры между процессами (SQLCipher-блокировки, дублирование LAN/P2P/Nostr listeners). Вкладки + горячие клавиши дают эквивалентный UX «chat simultaneously in multiple chats» без рисков.

## [2.5.26] — 2026-04-19

**Stealth-фильтр, Away auto-reply, умный поиск, jump-to-quote**

### Добавлено
- **Stealth broadcast-фильтр**: LAN unicast hello теперь пропускает контактов из `stealth_list`. `load_stealth_list()` хелпер в `lib.rs`. Контакт больше не видит нас онлайн если мы в «невидимке» для него.
- **Away auto-reply**: при статусе `away` / `dnd` на входящее сообщение автоматически отправляется настраиваемый ответ (кулдаун 10 мин на контакта). Два поля `away_message` / `dnd_message` в настройках (по умолчанию «Я отошёл — отвечу позже.» и «Не беспокоить — отвечу позже.»). UI: два textarea в `Settings.tsx`.
- **Умный поиск контактов** в Sidebar: `@xxx` → поиск по `custom_id`, 32+ hex символов → по `public_key` prefix, иначе → по nickname / local_alias / custom_id одновременно. Совпадение подсвечивается `<mark>`.
- **Цитирование с jump-to-message**: клик на цитате над сообщением прокручивает к оригиналу и подсвечивает его на 1.5 сек (`@keyframes message-jump-pulse` — жёлтая вспышка).

## [2.5.24] — 2026-04-19

**ICQ-функционал: Жужжалка, Auto-away, Экспорт чата**

### Добавлено
- **🔔 Жужжалка (ICQ Nudge)** — кнопка-колокольчик в заголовке 1-на-1 чата. Отправляет контакту пакет `nudge` одновременно по всем каналам (LAN / P2P / Nostr), у получателя окно трясётся (CSS `@keyframes nudge-shake`), играет низкий «бззз» (Web Audio sawtooth+square 110/87 Гц с 18 Гц LFO-модуляцией, `playBuzzSound` в `utils/sounds.ts`), показывается toast и системное уведомление если окно не в фокусе. Работает даже с другом за границей через Nostr-relays.
- **Auto-away** — после 5 минут без активности (mousemove/mousedown/keydown/wheel/touchstart) статус автоматически переключается в «Отошёл». При возврате активности (если `away` выставлен авто-режимом) возвращает «В сети». Уважает ручные выборы — не перезаписывает `dnd`/`na`/`invisible`.
- **Экспорт чата в .txt** — кнопка со стрелкой-вниз в заголовке чата выгружает всю историю в читаемый текстовый файл с шапкой (имя контакта, дата экспорта, количество сообщений) и строками формата `[дата время] Автор: текст`. Имя файла: `soviet-chat-<contact>-<timestamp>.txt`.

### Backend
- `make_nudge_packet` в `src-tauri/src/network/mod.rs` — строит `LanPacket` с `packet_type: "nudge"` и `payload.sender_pk`.
- `send_nudge` Tauri-команда в `lib.rs` — мультиканальная отправка (LAN + P2P + Nostr) с флагом `any_sent`, идентично send_message.
- `"nudge"` добавлен в `handle_lan_packet` — эмитит Tauri-событие `nudge` для фронта.
- `nostr.rs`: `"nudge"` добавлен в список Kind 4444 routed packet_types наряду с `"typing"`.

## [2.5.22] — 2026-04-18

**Popout удалён полностью + критический фикс Nostr DM (друг за границей теперь добавляется)**

### Удалено
- **Popout-окна для чатов и каналов** — функция «Открыть в новом окне» убрана целиком. На Windows WebView2 secondary-окна Tauri v2 показывали чёрный пустой экран независимо от подхода (popout.html, index.html+init_script, hash fragment) — корень проблемы в самом WebView2/Tauri и обходных путей не нашлось. Файлы `popout.html`, `PopoutRoot.tsx`, `popout.tsx`, `pages/ChatPopout.tsx` удалены. Rust-команды `open_chat_window`, `open_channel_window`, `get_popout_data` и `popout_registry` вычищены из `lib.rs` (~205 строк). Кнопки 🪟 убраны из `Sidebar`, `ChatWindow`, `ChannelWindow`, `ContactContextMenu`. **Будет заменено вкладками** в следующем релизе — табы открытых чатов в одном окне, надёжно работают на любой платформе.

### Исправлено критически
- **Контакт-запросы через Nostr DM не доходили до получателя** — корень: при первом запуске приложения identity ещё не создана, и Nostr-стартер захватывал пустую строку `my_soviet_pk_hex` в `move`-замыкания reader-task'ов каждого WebSocket'а. После онбординга `SubscribeDms` обновлял relay-side фильтр, но локальный обработчик `on_relay_msg` продолжал использовать **закэшированную пустую строку** и молча отбрасывал ВСЕ входящие Kind 4444 события через проверку `recipient_hex != my_soviet_pk_hex || my_soviet_pk_hex.is_empty()`. Теперь `on_relay_msg` читает актуальный `identity` из `AppState` на каждое событие — DM, contact requests, групповые приглашения через интернет наконец доходят. **Это блокировало возможность общаться с другом за границей через damus.io / nos.lol relays — теперь работает.**

## [2.5.21] — 2026-04-18

**Popout теперь через URL hash fragment — НЕ зависит от Tauri initialization_script**

### Исправлено
- **v2.5.20 по-прежнему чёрное окно** — ни зелёный баннер, ни watchdog из `initialization_script` не появлялись. Вывод: на Windows WebView2 для secondary-окон `initialization_script` в Tauri v2 **вообще не срабатывает**. Переделано радикально — popout-окна открываются с URL вида `index.html#popout=<label>`. Inline-скрипт прямо в `<head>` тегах `index.html` парсит `location.hash`, выставляет `window.__sovietPopoutLabel` и рисует зелёный статус-баннер с красной кнопкой «✕ Закрыть». Это обычный HTML — выполняется при парсинге страницы, не может «не сработать».
- **Watchdog 8 сек** — если React не смонтировался, inline-скрипт сам пишет в `#root` сообщение об ошибке с инструкцией открыть DevTools.
- **main.tsx fallback** — дополнительно парсит `location.hash` сам, на случай если inline-скрипт не успел выставить `window.__sovietPopoutLabel`.

## [2.5.20] — 2026-04-18

**Popout теперь ГАРАНТИРОВАННО не может быть чёрным пустым окном**

### Исправлено
- **Чёрное пустое popout-окно на v2.5.19** — даже с переездом на `index.html` секундарное Tauri-окно на Windows WebView2 отказывалось монтировать React (причина пока не ясна, предположительно timing asset protocol). Новый подход: `initialization_script` теперь **сам рисует UI напрямую** через `document.createElement` — зелёный статус-баннер с именем чата/канала + красная кнопка «✕ Закрыть». Этот UI появляется ДО любой попытки загрузить `main-*.js` и не зависит ни от React, ни от внешних ресурсов. React потом монтируется в `#root` ниже баннера, если успешно загружается.
- **Watchdog 8 секунд** — если через 8 сек React так и не появился в `#root`, init-script сам пишет в `#root` текст «React не смонтировался» + инструкцию открыть DevTools. Пользователь больше никогда не остаётся без визуальной обратной связи.
- **Безопасная кнопка закрытия** — встроена напрямую в баннер, использует `window.__TAURI__.window.getCurrentWindow().close()` с fallback на `window.close()`. Работает даже если JS-бандл полностью упал.

## [2.5.17] — 2026-04-18

**Архитектурный фикс popout-окон: чёрное/пустое окно больше не появляется**

### Исправлено
- **Popout-окно открывалось чёрным/пустым и не закрывалось** — на части Windows-конфигов WebView2 по неизвестной причине не рендерил отдельный `popout.html` (curl отдавал корректный HTML, WebView показывал чёрный экран). Сразу несколько попыток патчей (`.decorations(true)`, `.background_color()`, emergency-close-кнопка, watchdog-таймер) симптом не снимали — кнопка ✕ не появлялась, пользователь не мог закрыть окно. **Архитектурное решение:** popout-окна теперь используют тот же `index.html` и тот же React-bundle что и главное окно (которое работает гарантированно). Метка окна передаётся через `initialization_script` — `window.__sovietPopoutLabel = "chat-123"` выставляется до загрузки React; `main.tsx` проверяет флаг и рендерит `<PopoutRoot/>` вместо `<App/>`. Разница между главным и popout — один булев флаг в корне дерева.
- **Глобальные обработчики ошибок с fallback-рендерингом** — `window.addEventListener('error')` и `unhandledrejection` в `main.tsx` (активны только в popout режиме, чтобы не мешать React error boundary главного окна). Если React упал во время монтирования — пользователь видит сообщение «Ошибка загрузки окна» с работающей кнопкой «Закрыть» на инлайновых hex-цветах (`#0A0A0A`/`#F4FFF7`, не от CSS-переменных), гарантированно видимой даже если `global.css` не загрузился.

### Добавлено (инфраструктура — не в релизе)
- **`relay-server/`** — отдельный standalone libp2p-бинарник для self-hosted relay + rendezvous сервера. **Не публикуется в GitHub** (добавлен в `.gitignore`) — планируется поднять его на VPS/Raspberry Pi автора. Построен на libp2p 0.53 с behaviours: `relay::Behaviour`, `rendezvous::server::Behaviour`, `identify`, `ping`. Слушает TCP + QUIC на `:4001`, хранит Ed25519-ключ в `~/.soviet-relay/identity.key` (стабильный PeerId между рестартами).

### Для разработчиков
- **Сервер не нужно пересобирать при каждом релизе клиента** — `soviet-relay` отдельный бинарник с собственным `Cargo.toml`, не зависит от кода клиента. Протокол стабильный (`/libp2p/circuit/relay/0.2.0`, `/rendezvous/1.0.0`, identify `/soviet/relay/1.0.0`). Пересборка нужна только при изменении `relay-server/src/**`, bump libp2p, или смене rendezvous namespace.

## [2.5.14] — 2026-04-17

**Патч: исправлено зависание создания канала Nostr**

### Исправлено
- **«Создание...» висело долго, канал появлялся только после закрытия диалога** — корневая причина: runtime Nostr работал на `new_current_thread()` (одна нить для всего — reader-тасков, writer-тасков и command-loop). Когда приходила большая пачка входящих событий, обработчик держал блокирующий `db.lock().unwrap()` (`std::sync::Mutex`), блокируя всю нить; команды вроде `CreateChannel` ждали свою очередь. Перешли на **multi-thread runtime с 2 worker-потоками** — команды и обработка событий теперь независимы
- **Dead-relay мог подвесить запуск Nostr** — `connect_async` не имел таймаута, повисший TCP-коннект тянул минуты. Добавлен **5-сек таймаут на каждый relay**, битые relay пропускаются с логом
- **Дополнительная страховка на клиенте** — `nostr_create_channel` теперь имеет **3-сек таймаут** на ответ от Nostr-loop. Если команда всё-таки зависла — возвращаем понятную ошибку вместо вечного ожидания. DB-запись по-прежнему атомарна, подписка на канал отправляется fire-and-forget
