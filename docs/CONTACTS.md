# Система контактов

## Обзор

Soviet Messenger использует **полностью децентрализованную** систему контактов. Нет центрального сервера имён, нет глобального реестра. Каждый пользователь хранит свой список контактов **локально в SQLite**.

---

## Идентификация контакта

### Публичный ключ (основной идентификатор)

Каждый пользователь однозначно идентифицируется своим **публичным ключом Ed25519** (32 байта, Base58-кодированием):

```
Публичный ключ:  5KYZdUEo39C1shXXPRJ4FxHPCcMHbhcVVxjBkzmV9Mfx (52 символа)
Декодирование:   32 bytes hexadecimal
```

**Гарантии:**
- ✓ Уникален для каждого человека
- ✓ Не может быть изменён
- ✓ Невозможно подделать (без доступа к приватному ключу)
- ✓ Mathematically одноусловен (привязан к приватному ключу)

### Никнейм (отображаемое имя)

Никнейм — это **не уникальный идентификатор**. Два разных человека могут иметь одинаковый никнейм.

```
Два контакта в вашем списке:
├─ Anna K.  (pubkey: 5KYZdUEo39C1shXXPRJ4FxHPCcMHbhcVVxjBkzmV9Mfx)
└─ Anna K.  (pubkey: 3dq9KJzV5qpL8xXXvV4h1YXKLMoY6zS1nNPp8rRqT2V)

Уникален только публичный ключ!
```

### Custom ID (опционально)

Дополнительный уникальный идентификатор, выбираемый пользователем:

```
Формат: @myid
Требования:
- 3-10 символов
- Только буквы (a-z, A-Z), цифры (0-9), подчёркивание (_)
- Видно в QR-коде и share card
- Используется для удобства, не для криптографии
- Может быть пустым

Пример:
@polarbear
@anna.k
@bear_2024
```

### Локальный псевдоним (только вы видите)

Вы можете задать контакту **локальный псевдоним** — имя, которое видите только вы:

```
В вашем списке:
Контакт 1:
├─ Публичный ключ: 5KYZdUEo39C1shXXPRJ4FxHPCcMHbhcVVxjBkzmV9Mfx
├─ Никнейм (его): PolarBear
├─ Custom ID: @polarbear
└─ Локальный псевдоним (только у вас): "Петя из работы"

В чате показывается: "Петя из работы" (или никнейм, если нет псевдонима)
```

## Добавление контакта

### Способ 1: По публичному ключу

Пользователь делится своим ключом (через другой мессенджер, на визитке, устно):

```
1. Откройте "Добавить контакт"
2. Вставьте публичный ключ:
   5KYZdUEo39C1shXXPRJ4FxHPCcMHbhcVVxjBkzmV9Mfx
3. (Опционально) Задайте локальный псевдоним
4. Отправьте запрос
```

### Способ 2: QR-код (рекомендуется)

Каждый пользователь может показать QR-код своего профиля:

```
Format QR:
soviet://<base58-pubkey>?nick=<URL-encoded-nickname>&id=<custom-id>

Пример:
soviet://5KYZdUEo39C1shXXPRJ4FxHPCcMHbhcVVxjBkzmV9Mfx?nick=PolarBear&id=@polarbear

Сканирование → автоматическое заполнение формы добавления
```

### Способ 3: Share Card / Invite Card

Можно генерировать текстовую карточку:

```
┌─────────────────────────────┐
│  Soviet Messenger           │
├─────────────────────────────┤
│  PolarBear                  │
│  @polarbear                 │
│                             │
│  5KYZdUEo39C1shXXPRJ4FxHPCc │
│  MHbhcVVxjBkzmV9Mfx         │
│                             │
│  [📋 Скопировать]           │
│  [🔗 Поделиться]            │
└─────────────────────────────┘

Format: Soviet | PolarBear | 5KYZdUEo39C1shXXPRJ4FxHPCcMHbhcVVxjBkzmV9Mfx | @polarbear
```

### Способ 4: Обнаружение в LAN

Если оба пользователя в одной локальной сети, Soviet **автоматически** обнаруживает соседей через UDP broadcast:

```
┌──────────────────────────────┐
│ 📡 РЯДОМ В СЕТИ (2)          │
├──────────────────────────────┤
│ ❓ Unknown User 1            │
│    [+ Добавить в контакты]   │
│                              │
│ ❓ Unknown User 2            │
│    [+ Добавить в контакты]   │
└──────────────────────────────┘
```

Клик на "Добавить" → отправляется запрос на добавление.

## Запрос на добавление в контакты

При добавлении контакта отправляется зашифрованный запрос:

```
Пользователь A                 Пользователь B
      │                               │
      │── contact_request ───────────►│
      │ (имя A + его PK)              │
      │ (зашифровано E2E)             │
      │                               │
      │                      ┌────────┴───────┐
      │                      │ Уведомление:   │
      │                      │ "PolarBear хочет│
      │                      │  добавить вас" │
      │                      │ [Принять][Отклонить]
      │                      └────────┬───────┘
      │
      │◄── contact_accept ────────────│
      │    (если принято)
      │    (имя B + его PK)
      │
  Добавлен в контакты (статус: online/offline)
                             Добавлен в контакты
```

### До принятия запроса

- ❌ Пользователи **не могут** обмениваться личными сообщениями
- ✓ Могут участвовать в публичных Nostr каналах
- ✓ Могут взаимодействовать в групповых чатах (если приглашены)

### После принятия запроса

- ✓ Оба видят друг друга в списке контактов
- ✓ Могут отправлять личные сообщения
- ✓ Видят статус друг друга (online/away/na/dnd/invisible/offline)
- ✓ Получают read receipts

## Структура контакта в БД

```sql
CREATE TABLE contacts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    public_key TEXT NOT NULL UNIQUE,    -- Base58 Ed25519 публичный ключ
    nickname TEXT NOT NULL,             -- Никнейм пользователя
    custom_id TEXT,                     -- @myid (опционально)
    local_alias TEXT,                   -- Локальный псевдоним (видит только вы)

    -- Статус
    status TEXT DEFAULT 'offline',      -- online | away | na | dnd | invisible | offline
    status_text TEXT,                   -- "На работе", "Обеденный перерыв", etc
    last_seen INTEGER,                  -- Unix timestamp последнего визита

    -- Аватар и метаданные
    avatar BLOB,                        -- PNG/JPEG в бинарной форме
    notes TEXT,                         -- Ваши личные заметки о контакте

    -- Управление
    is_blocked INTEGER DEFAULT 0,       -- 1 = заблокирован
    is_favorite INTEGER DEFAULT 0,      -- 1 = в избранном

    -- Время
    added_at INTEGER NOT NULL,          -- Unix timestamp добавления
    verified INTEGER DEFAULT 0,         -- 1 = fingerprint верифицирован
    verified_at INTEGER                 -- Unix timestamp верификации
);
```

## Статусы пользователя

| Статус | Иконка | Описание |
|--------|--------|----------|
| **Online** | 🟢 | Активен прямо сейчас |
| **Away** | 🟡 | Нет активности > 5 минут (автоматически) |
| **Busy** | 🔴 | Установлен вручную ("Не беспокоить") |
| **Invisible** | ⚫ | Выглядит как offline для других |
| **Offline** | ⚪ | Приложение закрыто / отключено |

### Автоматический переход в Away

```
Timeline:
0:00  ← пользователь активен (печатает, кликает)
5:00  ← неактивен > 5 минут → статус: away
10:00 ← вернулась активность → статус: online (снова)
```

### Пользовательский текст статуса

Каждый статус может иметь дополнительный текст:

```
🟢 Online — "Работаю над проектом"
🟡 Away — "Обеденный перерыв, вернусь в 14:00"
🔴 Busy — "На встрече, не беспокоить до 18:00"
⚪ Offline
```

### Сортировка контактов

По умолчанию:
1. Онлайн (🟢) в первую очередь, по алфавиту
2. Away (🟡) во вторую очередь, по алфавиту
3. Оффлайн (⚪) в конце, по алфавиту

Опционально можно переключиться на:
- Только по алфавиту (без группировки по статусу)
- По времени последней активности (новые вверху)

## Блокировка контактов

При блокировке пользователя:

```
Эффекты:
├─ ❌ Заблокированный НЕ может отправить вам сообщение
├─ ❌ Вы НЕ отображаетесь в его списке "Рядом" (LAN/Mesh)
├─ ✓ Заблокированный НЕ получает уведомления о блокировке
├─ ✓ История сообщений сохраняется (но скрывается в UI)
└─ ✓ Можно разблокировать в любой момент
```

## Верификация контакта (Safety Numbers)

Для защиты от MITM-атак (подмена публичного ключа) можно верифицировать контакт:

```
┌─────────────────────────────────────────┐
│  Профиль контакта: Anna K.              │
├─────────────────────────────────────────┤
│                                         │
│  Safety Number:                         │
│  ┌─────────────────────────────────────┐│
│  │ 12345 67890 12345 67890 12345 67890 ││
│  │ 12345 67890 12345 67890 12345 67890 ││
│  └─────────────────────────────────────┘│
│                                         │
│  [✓] Верифицирован лично              │
│                                         │
│  Сравните эти числа с вашим контактом  │
│  лично, по телефону или другому        │
│  зашифрованному каналу.                │
│  Если совпадают → ставьте галочку.     │
└─────────────────────────────────────────┘
```

**Как вычисляется Safety Number:**

```
safety_number = SHA256(sort(my_pubkey, contact_pubkey))
             → отображается как 12 групп по 5 цифр
             → 120 цифр всего (очень сложно подделать)
```

### Верифицированный контакт

После верификации контакт отмечается значком ✓:

```
✓ Anna K.  [Верифицировано 2024-04-02]
```

**Если Safety Number не совпадает:**
→ Возможна MITM-атака, **не продолжайте переписку**, свяжитесь с контактом по другому каналу.

## Экспорт и импорт контактов

### Экспорт

Все контакты экспортируются в зашифрованный файл `.sovietcontacts`:

```json
{
  "version": 1,
  "exported_at": 1700000000,
  "encrypted": true,
  "contacts": [
    {
      "public_key": "5KYZdUEo39C1shXXPRJ4FxHPCcMHbhcVVxjBkzmV9Mfx",
      "nickname": "PolarBear",
      "custom_id": "@polarbear",
      "local_alias": "Петя из работы",
      "notes": "Познакомились на конференции"
    },
    ...
  ]
}
```

**Процесс:**
1. Клик [Экспорт контактов]
2. Система просит пароль (опционально)
3. Генерируется файл `contacts_2024-04-02.sovietcontacts`
4. Загружается на компьютер

### Импорт

При импорте файла контактов:

```
Импорт contacts_2024-04-02.sovietcontacts

1. Загружается файл
2. Система проверяет версию и формат
3. Показывает превью: "20 контактов для импорта"

   Что будет:
   ├─ 15 новых контактов (добавляются)
   ├─ 5 существующих (не перезаписываются)
   └─ 0 конфликтов

4. [Импортировать] или [Отмена]
```

**Гарантии:**
- ✓ Новые контакты добавляются
- ✓ Существующие контакты **не перезаписываются**
- ✓ История сообщений сохраняется
- ✓ Можно импортировать несколько раз (дубликаты игнорируются)

## Аватары

### Установка аватара

```
┌─────────────────────────────────────┐
│  Аватар профиля                     │
├─────────────────────────────────────┤
│                                     │
│          [Текущий аватар]           │
│          или идентикон              │
│                                     │
│  [📁 Выбрать файл] (PNG/JPEG)       │
│  [🗑️ Удалить аватар]               │
│                                     │
│  Рекомендации:                      │
│  - Максимум 256x256 px              │
│  - JPG/PNG                          │
│  - < 100 КБ                         │
└─────────────────────────────────────┘
```

### Хранение и синхронизация

- **Ваш аватар** → сохраняется в `settings` (base64)
- **Аватары контактов** → хранятся в БД (BLOB)
- **Обновление** → при следующем контакте / в Nostr профиле
- **Синхронизация** → **не синхронизируется** между устройствами (v1.0)

### Идентиконы

Если аватар не установлен — генерируется **визуальный хэш** (идентикон):

```
Идентикон = детерминированное изображение, основанное на хэше публичного ключа

Свойства:
├─ Разные для каждого публичного ключа
├─ Одинаковые при запуске (детерминированные)
├─ Цветные, геометрические
└─ Помогают визуально узнать контакт
```

## Синхронизация между устройствами

**В v1.0**: Синхронизация **не поддерживается**.

Каждое устройство имеет:
- Свой приватный ключ (уникальный для этого устройства)
- Свой список контактов (локально)
- Свою историю сообщений (локально)

**Рекомендация для пользователей:**
- Экспортируйте контакты на одном устройстве
- Импортируйте на другом
- История не восстанавливается (начнётся с нуля)

**Планируется в v2.0:**
- Синхронизация через зашифрованный облачный хранилище (Dropbox, iCloud, etc.)
- Multi-device sync через Nostr
- Одноустройственная синхронизация (одна личность на нескольких девайсах)
